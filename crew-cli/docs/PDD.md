# crew-cli — Product Design Document

> Generated by CrewSwarm on 2026-02-28. This is the definitive design document for the multi-agent coding CLI.

## 1. Problem Statement

**What we're building:** A terminal-based AI coding CLI that orchestrates multiple specialist agents and existing CLI tools (Aider, Gemini, Codex, Claude Code) to provide superior coding assistance with cost optimization, safety, and intelligence.

**Who it's for:**
- Professional developers working on large codebases
- Teams that need consistent, safe AI-assisted development
- Users with Claude Pro/Cursor Pro subscriptions who want to maximize their unlimited credits
- Developers who need multi-agent collaboration (PM, QA, security, etc.) not just code editing

**The problem:**
- **Existing CLIs are single-purpose:** Aider only edits, Gemini only uses Google models, Codex only uses OpenAI
- **No safety:** Changes apply immediately to real files with no review sandbox
- **No intelligence:** Can't automatically pick the best tool/model for each task
- **Expensive:** Users burn through API credits when cheaper/unlimited options exist
- **No multi-agent:** Can't dispatch to specialists (QA, security, PM) from the same interface
- **Context loss:** CLIs forget everything between runs

**Why now:**
- 6 major coding CLIs exist (Aider 41k⭐, Cursor 32k⭐, Codex 62k⭐, Gemini 96k⭐, Claude Code 71k⭐, Plandex 15k⭐) but none orchestrate all of them
- CrewSwarm already has 20 specialist agents working autonomously
- OAuth token reuse can provide unlimited access to premium subscriptions
- All major CLIs are open source (Apache 2.0 / MIT) so we can vendor their best parts

## 3. Scope

### In scope (Phase 1 - MVP)
- ✅ Git context auto-injection (branch, status, diff, commits)
- ✅ Dual-LLM orchestrator (Groq for routing, expensive models for execution)
- ✅ Integration with CrewSwarm gateway (dispatch to 20 agents)
- ✅ OAuth token finder (reuse Claude/Cursor/Gemini sessions)
- ✅ 4 vendored edit strategies from Aider (editblock, unified-diff, whole-file, search-replace)
- ✅ Cumulative diff sandbox from Plandex (changes don't touch real files until approved)
- ✅ Plan-first workflow (show plan, get approval, execute)
- ✅ Subprocess calls to Gemini/Codex/Claude CLIs when installed
- ✅ Session state (`.crew/session.json` for chat history, cost tracking)
- ✅ Basic TUI (streaming, syntax highlighting, spinners)
- ✅ Installer script (checks/installs dependencies, finds OAuth tokens)

### In scope (Phase 2 - Intelligence)
- ✅ Speculative execution (3 parallel approaches, user picks best)
- ✅ Cost prediction (estimate before execution, offer alternatives)
- ✅ Learning from corrections (build local LoRA training data)
- ✅ Branch/compare alternatives (like Plandex branches)
- ✅ Automated debugging (run linters/tests, auto-fix)
- ✅ Watch mode (monitor files, implement TODOs in comments)

### Out of scope (for now)
- ❌ GUI/desktop app (terminal only for MVP)
- ❌ Cloud hosting (local-first, connects to existing CrewSwarm)
- ❌ Training custom models (just uses existing LLMs)
- ❌ IDE plugins (CLI-first, IDE integration later)
- ❌ Windows native support (WSL only initially)

## 5. Technical Architecture

### High-level design
```
┌─────────────────────────────────────────────────┐
│           crew CLI (TypeScript)                 │
│  Entry point, TUI, session management           │
└────────────┬────────────────────────────────────┘
             │
┌────────────▼────────────────────────────────────┐
│       Orchestrator (Groq llama-3.3-70b)         │
│  Cheap, fast routing: CHAT | CODE | DISPATCH    │
└────┬───────┬────────┬───────────────────────────┘
     │       │        │
     │       │        └─> CrewSwarm Gateway (:5010)
     │       │            ├─> crew-pm (planning)
     │       │            ├─> crew-coder (code)
     │       │            ├─> crew-qa (tests)
     │       │            └─> crew-security (audit)
     │       │
     │       └─────────> Code Executors:
     │                   ├─> Vendored strategies (editblock, diff, etc)
     │                   ├─> Gemini CLI (subprocess)
     │                   ├─> Codex CLI (subprocess)
     │                   ├─> Claude Code CLI (subprocess)
     │                   └─> Direct API (Grok, DeepSeek, etc)
     │
     └──────────────────> Cumulative Diff Sandbox
                         ├─> Branches (explore alternatives)
                         ├─> Preview (show all changes)
                         ├─> Apply (write to real files)
                         └─> Rollback (undo)
```

### Key components

#### 1. Git Context Reader
```typescript
// src/context/git.ts
export async function getProjectContext(cwd: string): Promise<Context> {
  return {
    branch: await exec('git branch --show-current'),
    status: await exec('git status --short'),
    diff: await exec('git diff'),
    staged: await exec('git diff --staged'),
    commits: await exec('git log -5 --oneline')
  };
}
```

#### 2. Orchestrator (Dual-LLM)
```typescript
// src/orchestrator.ts
export async function route(message: string, context: Context) {
  // Cheap model decides what to do
  const decision = await groq.chat({
    system: 'Route to: CHAT, CODE, SKILL, or DISPATCH',
    user: message,
    context
  });
  
  if (decision.action === 'CODE') {
    // Expensive model writes code
    return await executeCode(message, decision.strategy);
  } else if (decision.action === 'DISPATCH') {
    // Send to CrewSwarm agent
    return await crewswarm.dispatch(decision.agent, message);
  }
  // ... etc
}
```

#### 3. Sandbox
```typescript
// src/sandbox/index.ts
export class Sandbox {
  files: Map<string, FileState>;
  branches: Branch[];
  
  async addChange(path: string, change: Change) {
    // Accumulate, don't apply to real file
  }
  
  async preview(): string {
    // Generate unified diff
  }
  
  async apply() {
    // Write all changes to real files
  }
}
```

#### 4. OAuth Token Finder
```typescript
// src/auth/token-finder.ts
export async function findTokens() {
  const tokens = {};
  
  // Check ~/.claude/session.json
  if (existsSync('~/.claude/session.json')) {
    tokens.claude = await extractClaudeToken();
  }
  
  // Check ~/.cursor/ SQLite DB
  if (existsSync('~/.cursor/User/globalStorage/state.vscdb')) {
    tokens.cursor = await extractCursorToken();
  }
  
  return tokens;
}
```

```

  ↓
```

## 7. Non-Functional Requirements

### Performance
- **Startup time:** < 500ms (TypeScript, no heavy deps)
- **Context injection:** < 100ms (git commands are fast)
- **Orchestrator decision:** < 2s (Groq @ 2000 tok/s)
- **Streaming:** Real-time (SSE from LLMs)

### Security
- **OAuth tokens:** Read-only, never transmitted, stored in OS keychain where possible
- **API keys:** Inherit from ~/.crewswarm/crewswarm.json (user's responsibility)
- **Sandbox:** Changes isolated until user approval
- **Command execution:** Only via approved engines (no arbitrary shell injection)

### Reliability
- **Offline graceful:** Work without CrewSwarm if gateway unavailable (local engines only)
- **Rollback:** Every change is reversible via sandbox
- **Error handling:** Catch subprocess crashes, show helpful messages
- **Telemetry:** Opt-in anonymous usage stats

### Cost
- **Orchestrator:** $0.59/M tokens (Groq)
- **Typical task:** < $0.10 (orchestrator + cheap executor)
- **OAuth users:** $0 (unlimited via existing subscriptions)
- **Target:** 10x cheaper than pure Claude API usage

## 8. Testing Strategy

### Unit tests
- Git context parser (mock git commands)
- Orchestrator routing logic (mock LLM responses)
- Sandbox operations (file accumulation, diff generation)
- OAuth token extraction (mock file system)

### Integration tests
- Full flow: user input → orchestrator → executor → sandbox → apply
- CrewSwarm gateway dispatch (requires :5010 running)
- Subprocess calls (Gemini/Codex/Claude if installed)

### E2E tests
- Real project scenarios (add auth, refactor module, etc.)
- Multi-agent workflows (PM → coder → QA → github)
- Speculative execution (3 parallel branches)
- Cost tracking accuracy

### Manual testing
- Test on 3 real codebases (small/medium/large)
- Test with different API key combinations
- Test OAuth token reuse (Claude Pro, Cursor Pro)
- Test with/without optional engines installed
