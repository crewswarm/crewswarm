# Positioning

Updated: March 14, 2026

Use this when explaining crewswarm to employers, investors, sponsors, or technical users.

## One-line Positioning

crewswarm is the local-first multi-agent dev stack: orchestration, coding surfaces, memory, and service control in one system.

## Short Pitch

crewswarm turns AI-assisted coding from a single chat box into an actual operating stack.

Instead of one editor plugin or one agent loop, it gives you:
- a real orchestration runtime
- multiple specialist agents
- multiple operator surfaces
- project-aware history and memory
- local control over services, models, and execution paths

## What makes it different

- not just prompts: real services, routes, state, and control surfaces
- not just one model: per-agent runtime and model routing
- not just one UI: Dashboard, Vibe, `crewchat`, SwiftBar, MCP
- not cloud-dependent: local-first and self-hostable
- not fake file edits: actual disk IO and project-aware flows

## Who it is for

- solo builders who want a stronger AI dev environment
- technical founders building with agent workflows daily
- small AI-native product teams
- local-first users who want visibility and control
- developers building custom agent infrastructure on top of a real stack

## Job Framing

What crewswarm proves:
- systems design
- orchestration/runtime engineering
- product thinking across multiple surfaces
- frontend + backend + tooling breadth
- pragmatic AI integration beyond toy demos

Good framing:
- “Built a local-first multi-agent software development platform with real orchestration, memory, IDE/chat surfaces, and editor integrations.”

## Investor Framing

Useful framing:
- local-first AI developer operating system
- orchestration layer for multi-agent software work
- control plane for AI-native builders and teams

Questions this project answers well:
- can the product actually run end-to-end?
- is there a wedge beyond chat?
- is there technical depth?
- can this become a platform, not just a feature?

## Sponsor Framing

What sponsors care about:
- real usefulness
- active maintenance
- credible documentation
- visible roadmap
- a project that helps serious users do real work

Good sponsor framing:
- “Support the development of a self-hostable, local-first multi-agent coding stack.”

## Honest Boundaries

Do not oversell:
- UX polish is improving, not perfect
- universal one-command deployment for every generated app is not the current claim
- some surfaces are more mature than others

Best honest claim:
- strong public release with real technical depth, real surfaces, and real installability
